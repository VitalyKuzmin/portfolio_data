# Процесс разработки и CI/CD: Инвентаризация

## 1. Философия
Процесс был адаптирован под гибридную систему с разнородными компонентами (1С, outsourced Android, собственный Electron). Фокус был на **ручном интеграционном тестировании**, так как полная автоматизация была нецелесообразна.

## 2. Стек инструментов
- **Репозиторий:** GitLab (только для Electron-приложения).
- **CI/CD:** GitLab CI (только для сборки Electron).
- **Тестирование:** TestRail для ведения тест-кейсов.

## 3. Процесс разработки и сборки

### Компонент 1: Android-приложение
- Разрабатывалось внешним подрядчиком.
- Поставка осуществлялась в виде готового `.apk` файла раз в 2-4 недели.

### Компонент 2: Бэкенд 1С
- Разрабатывался штатным 1С-разработчиком.
- Обновления конфигурации (`.cf`) выкатывались на тестовый, а затем на продуктовый сервер по мере готовности.

### Компонент 3: Desktop-приложение (Electron)
- Разработка велась в GitLab.
- Был настроен простой CI-пайплайн в GitLab CI:
    1.  **`lint`**: Проверка кода с помощью `eslint`.
    2.  **`build`**: Сборка приложения под Windows с помощью `electron-builder`.
    3.  **`package`**: Создание `.exe` инсталлятора.
- Артефакты (инсталляторы) сохранялись в GitLab.

## 4. Тестирование и Релизный цикл

Релизный цикл был синхронизирован для всех компонентов и выглядел так:

1.  **Планирование (1-2 дня):** Я, как руководитель проекта, совместно с 1С-разработчиком и представителем подрядчика планировал объем работ на следующий релизный цикл (обычно 1 месяц).
2.  **Разработка (3 недели):** Команды параллельно вели разработку своих частей.
3.  **Интеграционное тестирование (1 неделя):**
    - QA-инженер получал последние сборки всех трех компонентов.
    - На отдельном **Staging-сервере** (копия прода) разворачивалась новая конфигурация 1С.
    - QA устанавливал на тестовые устройства `.apk` и `.exe` клиенты.
    - Проводилось полное **сквозное (E2E) тестирование** по заранее написанным в TestRail сценариям: от создания объекта в мобильном приложении до его появления на карте в десктопе и генерации отчета.
4.  **Релиз:**
    - После успешного прохождения всех тестов назначался день релиза.
    - Обновление всех компонентов производилось **вручную** в согласованное время.
