# Оценка нагрузки и ресурсов: Инвентаризация

Это внутренний корпоративный инструмент, поэтому нагрузка низкая, предсказуемая и ограниченная количеством сотрудников.

## 1. Исходные допущения

- **Количество пользователей:** 20 полевых сотрудников + 5 инженеров в офисе.
- **Интенсивность работы:**
    - Каждый полевой сотрудник обрабатывает ~50 объектов в день.
    - Каждый объект = ~10 полей метаданных + 2 фотографии (~1 MB суммарно).
- **Синхронизация:** Полевые сотрудники синхронизируют данные с сервером 3-4 раза в день.

## 2. Расчет нагрузки

| Метрика | Расчет | Значение |
| :--- | :--- | :--- |
| **Объектов в день** | 20 пользователей * 50 объектов | 1,000 |
| **Объем данных в день** | 1,000 объектов * 1 MB | **~1 GB / день** |
| **API сессий в день** | 20 пользователей * 4 синхронизации | **~80** |
| **Пиковый RPS** | - | **Неприменимо.** Нагрузка не измеряется в RPS, а в количестве и объеме синхронизаций. |

## 3. Оценка ресурсов

- **"Узкое место":** Единственным узким местом является **сервер 1С** и его способность обрабатывать входящие HTTP-запросы с файлами.
- **Инфраструктура:** Не требуется никакой дополнительной инфраструктуры. Нагрузка полностью ложится на существующий сервер 1С.

**Итог:** Нагрузка на систему минимальна. Ключевой задачей является обеспечение стабильности HTTP-сервисов на стороне 1С и надежности сетевого подключения для полевых бригад.
