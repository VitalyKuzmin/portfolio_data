# Оценка нагрузки и ресурсов: Онлайн-калькуляторы (Galad.ru)

## 1. Исходные допущения

- **Ежедневные посетители:** 50,000
- **Пиковые часы:** 6 часов, на которые приходится 60% трафика.
- **Среднее количество действий:** 5 запросов на пользователя (изменение параметров, расчеты).
- **Генерация PDF:** 5% посетителей создают один отчет.

## 2. Расчет нагрузки (RPS и Задачи)

| Метрика | Расчет | Значение |
| :--- | :--- | :--- |
| **Всего API запросов в день** | 50,000 * 5 | 250,000 |
| **Пиковый RPS (API Gateway)** | (250,000 * 0.6) / (6*3600) | **~7 RPS** |
| **Пиковый RPS (Calc Service)**| ~7 RPS * 0.2 (каждый 5-й запрос - расчет) | **~1.5 RPS** (но CPU-heavy) |
| **Задач на генерацию PDF** | 50,000 * 0.05 | **~2,500 / день** |

## 3. Расчет хранения данных

- **PostgreSQL (Проекты):**
    - Предположим, 10% пользователей сохраняют проекты: 5,000 проектов в день.
    - При среднем размере проекта 10 KB, это **~50 MB / день** дискового пространства. Нагрузка на запись невысокая.

## 4. Оценка ресурсов (Инфраструктура на VM)

Поскольку система не такая сложная, как Jarvel, можно обойтись без Kubernetes.

| Компонент | Ресурсы | Примечание |
| :--- | :--- | :--- |
| **API Gateway (Nginx)** | 1 VM (2 CPU, 2 GB RAM) | Стандартная конфигурация для проксирования. |
| **Python Services** | 2 VM (2 CPU, 4 GB RAM) | Запуск контейнеров с Project Service, API для отчетов и т.д. |
| **Calculation Service** | **1-2 VM (4-8 CPU, 8 GB RAM)** | **"Узкое место".** Требует много CPU из-за вызовов C++ библиотек. Масштабируется увеличением числа Docker-контейнеров на этих VM. |
| **Celery Workers (PDF)** | 1 VM (4 CPU, 4 GB RAM) | Для фоновой обработки тяжелых задач генерации отчетов. |
| **PostgreSQL / Redis** | 1 VM (4 CPU, 8 GB RAM) | Стандартная конфигурация для базы данных и кэша. |

**Итог:** Ключевой вызов — обеспечение достаточных CPU-ресурсов для расчетного сервиса. Остальная инфраструктура имеет умеренные требования.
