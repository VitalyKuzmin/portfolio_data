# Архитектура проекта: Онлайн-сервисы (Galad.ru)

## 1. Обзор задачи
**Контекст:** Существующий корпоративный сайт работает на **1С-Битрикс (PHP + MySQL)**. Это монолит, который сложно и дорого дорабатывать для сложной математики.
**Цель:** Внедрить интерактивные калькуляторы и личный кабинет с расчетами, не переписывая весь сайт, но используя современные технологии (Python) для вычислений.

---

## 2. Схема интеграции (Strangler Pattern)

Мы использовали гибридный подход. Битрикс остается "витриной" и провайдером авторизации, а сложная логика вынесена в микросервисы.


---

## 3. Микросервисы (Детализация)

Всего было выделено **4 основных микросервиса**, упакованных в Docker:

### 1. Calculation Core (Ядро расчетов)
*   **Роль:** Обертка над старыми C++ библиотеками.
*   **Техника:** Использовал `ctypes` / `subprocess` для вызова бинарников C++ из Python.
*   **Почему отдельно:** C++ код мог упасть (Segmentation Fault) при кривых данных. Изоляция в контейнере защищала остальной сайт от падения. Плюс можно масштабировать (поднять 5 контейнеров при нагрузке).

### 2. Project Manager (Личный кабинет)
*   **Роль:** CRUD операции. Сохранение проектов пользователей, списка помещений, выбранных светильников.
*   **Стек:** FastAPI + PostgreSQL.
*   **Особенность:** Хранил JSON-структуру проекта пользователя.

### 3. Report Service (Генератор отчетов)
*   **Роль:** Асинхронная генерация PDF (Светотехнический отчет).
*   **Стек:** Celery Worker + библиотекa для PDF (WeasyPrint или ReportLab).
*   **Процесс:** Тяжелая операция, поэтому вынесена в фон. Пользователь нажимает "Скачать", задача летит в очередь (Redis), фронт опрашивает статус.

### 4. Catalog Sync (Синхронизация данных)
*   **Проблема:** Актуальные цены и параметры светильников лежат в MySQL Битрикса.
*   **Решение:** Сервис раз в сутки (или по триггеру) забирал данные из Битрикса и обновлял локальную базу PostgreSQL калькуляторов, чтобы расчеты велись по актуальным IES-файлам.

---

## 4. Frontend и Встраивание
*   **Технология:** React-компоненты (SPA), встраиваемые внутрь страниц Битрикса.
*   **Авторизация:**
    1.  Пользователь логинится в Битрикс (PHP сессия).
    2.  Битрикс генерирует временный токен (JWT) и передает его во фронтенд-компонент.
    3.  React ходит в Python API с этим токеном.
    4.  Python API валидирует токен (общий секрет с Битриксом).

---

## 5. Почему это сработало (Business Value)
1.  **Скорость:** Менеджеры продаж перестали ждать инженеров. Простой расчет делается за 2 минуты прямо при клиенте.
2.  **Надежность:** Если расчет завис, основной сайт Galad.ru продолжает работать.
3.  **Time-to-market:** Мы могли выкатывать обновления калькуляторов хоть каждый день, не трогая неповоротливый релизный цикл Битрикса.
