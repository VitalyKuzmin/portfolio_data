# Архитектура проекта: МСЛ Картирование (Mobile Lighting Laboratory)

## 1. Обзор системы
Программно-аппаратный комплекс для сбора, синхронизации и визуализации гео-привязанных данных светотехнических измерений.
Система решает проблему обработки "сырых" данных с разнородных датчиков мобильной лаборатории и предоставления их в удобном виде для анализа освещенности городов.

---

## 2. Компоненты системы

### A. Hardware Layer (Сбор данных)
На борту автомобиля установлены датчики с разной частотой опроса:
*   **Люксметр:** Измеряет освещенность (высокая частота, ~100 Гц).
*   **GPS-приемник:** Координаты и время (низкая частота, 1 Гц).
*   **Датчик пути (Одометр):** Точная скорость и пройденное расстояние.

**Проблема:** Рассинхронизация данных. У нас есть много измерений света между двумя точками GPS.
**Решение:** Данные пишутся в сырые бинарные логи, постобработка происходит на клиенте.

### B. Desktop Processing App (C++ / MFC)
"Толстый клиент" для инженера.
*   **Стек:** C++, MFC (Microsoft Foundation Classes).
*   **Задачи:**
    1.  **Парсинг:** Чтение бинарных логов с устройств.
    2.  **Синхронизация (Core Logic):** Алгоритм интерполяции координат. На основе времени и скорости вычисляется точное положение автомобиля для каждого замера освещенности (между точками GPS).
    3.  **Фильтрация:** Устранение шумов GPS и случайных засветок.
    4.  **Экспорт:** Передача обработанного массива данных в 1С. Использовалась **DLL-библиотека** для нативного вызова методов платформы 1С или COM-соединение.

### C. Storage Layer (1С:Предприятие)
*   **Роль:** Основное хранилище данных (Master Data).
*   **Почему 1С:** Требование заказчика/интеграция с существующими процессами учета.
*   **Функции:** Хранение справочников улиц, городов и привязка к ним массивов измерений.

### D. Visualization Layer (Web)
*   **Backend:** PHP. Выполняет роль прослойки (Proxy). Запрашивает данные из 1С (через OData или SQL) и отдает на фронт.
*   **Frontend:** JavaScript, Bootstrap.
*   **Maps API:** Yandex Maps API (выбран из-за лучшей детализации дорог в РФ).
*   **Функционал:**
    *   Отрисовка трека движения.
    *   **Heatmap (Тепловая карта):** Цветовое кодирование точек трека в зависимости от яркости (зеленый — норма, красный — темно).
    *   Кластеризация точек при отдалении карты для оптимизации производительности.

## 3. Ключевые технические решения

1.  **Алгоритм интерполяции:**
    Реализован математический аппарат для привязки быстрых измерений датчиков к медленному GPS-треку. Это позволило получить плавную картину освещенности, а не "рваные" точки раз в секунду.

2.  **Оптимизация рендеринга:**
    При отображении данных целого города браузер не справлялся с отрисовкой десятков тысяч маркеров. Была применена кластеризация и упрощение геометрии линий при малом масштабе.

3.  **Интеграция C++ и 1С:**
    Написание внешней компоненты (Native API / COM) для прямой загрузки больших массивов данных из C++ приложения в базу 1С, минуя медленные текстовые обмены.
