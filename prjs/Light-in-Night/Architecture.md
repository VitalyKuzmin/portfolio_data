# Архитектура проекта: Light in Night Pro

## 1. Обзор системы
Профессиональная САПР (CAD) для светотехнического проектирования.
**Ключевая особенность:** Модульная архитектура "толстого клиента" с поддержкой облачных сервисов для лицензирования и аналитики.

---

## 2. Клиентская часть (Desktop)

Приложение реализовано на **C++** с использованием фреймворка **Qt** для кроссплатформенности (Windows / Linux — требование импортозамещения).

### Модульная структура (Plugin/DLL Architecture)
Ядро приложения управляет набором динамических библиотек (DLL / SO), что позволяло параллелить разработку разными командами:

1.  **Core UI (Qt):** Основной интерфейс, дерево проекта, свойства объектов.
2.  **Graphic Engine (OpenGL):**
    *   Отвечает за отрисовку 3D-сцены во вьюпорте (CAD-режим).
    *   *Исторический контекст:* Использовался Legacy OpenGL, который приходилось поддерживать и оптимизировать.
3.  **Render Engine (CUDA):**
    *   Отвечает за фотореалистичную визуализацию (Ray Tracing).
    *   Использует мощности GPU (Nvidia CUDA) для расчета отражений, теней и материалов.
4.  **Calculation Engine (C++):**
    *   Математическое ядро расчета освещенности.
    *   Разрабатывалось отдельной научной командой (профессора МЭИ).
    *   Отличается высокой надежностью и покрытием Unit-тестами.
5.  **Report Engine:**
    *   Генерация документации по ГОСТ.
    *   Разрабатывалось отдельным подрядчиком (аутсорс).

---

## 3. Серверная инфраструктура (Backend Services)

Хотя приложение десктопное, оно плотно интегрировано с веб-сервисами (Python/Go). За эту часть отвечал отдельный Backend-разработчик под моим руководством.

### Сервисы (Microservices):
1.  **Licensing Service (Guardant):**
    *   Управление лицензиями, защита от пиратства.
    *   Интеграция с аппаратными и программными ключами Guardant.
2.  **Telemetry & Logging (ELK Stack):**
    *   Приложение отправляет логи и краш-дампы в **Logstash**.
    *   Хранение в **Elasticsearch**, просмотр в **Kibana**.
3.  **Analytics (Grafana):**
    *   Визуализация продуктовых метрик.
    *   *Метрики:* DAU/MAU, частота использования инструментов, среднее время расчета, статистика по "железу" пользователей (GPU/CPU).
4.  **Catalog Sync Service:**
    *   Сервис актуализации базы светильников.
    *   Синхронизирует локальную базу SQLite клиента с мастер-базой ("Старт").
