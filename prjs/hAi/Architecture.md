# Архитектура проекта: hAi (Desktop AI Assistant)

## 1. Обзор системы

**hAi** — это локальный AI-ассистент для десктопа, построенный по принципу **распределенной локальной системы**. Его ключевая задача — помогать пользователю, обладая глубоким контекстом его действий, истории и персональных данных, при этом сохраняя всю информацию локально для обеспечения приватности.

Система состоит из нескольких независимых, но взаимосвязанных процессов:
- **Frontend (Electron):** Пользовательский интерфейс.
- **Backend (Python/FastAPI):** Ядро логики, управление AI-агентами.
- **Voice System (LiveKit):** Голосовое взаимодействие в реальном времени.
- **Activity Monitoring (ActivityWatch):** Сбор контекста о действиях пользователя.
- **MCP System:** Протокол для безопасного расширения функционала через плагины.

## 2. Компоненты системы

### A. Frontend (Electron)
- **Стек:** Electron, React, TypeScript.
- **Роль:** Отвечает за весь пользовательский интерфейс, включая оверлей чата, окно настроек и системный трей.
- **`main.js`:** Основной процесс Electron, который управляет жизненным циклом приложения и запускает Python-бэкенд как дочерний процесс.
- **`preload.js`:** Безопасный мост (IPC) между UI (Renderer) и системными вызовами в `main.js`.

### B. Backend (Python/FastAPI)
- **Стек:** Python, FastAPI, LangChain, SQLAlchemy.
- **Роль:** Ядро всей системы.
- **`main.py` (Orchestrator):** Запускает и координирует все остальные компоненты: FastAPI-сервер, LiveKit, ActivityWatch.
- **API Routers:** Предоставляют эндпоинты для чата, настроек и обмена состоянием с UI.
- **Agent Logic (LangChain):** Основной конвейер обработки запроса. Собирает контекст из всех источников (память, снэпшоты), выбирает инструменты и запускает LLM-агента (ReAct-подход).

### C. Система Памяти (Hybrid Memory Architecture)
Память — ключевой компонент, обеспечивающий контекстуальность.
- **Краткосрочная память (Short-Term):** Последние сообщения и действия пользователя хранятся в "сыром" виде для максимальной точности.
- **Эпизодическая память (Episodic):** Старые диалоги и события активности сжимаются в **саммари** с помощью LLM, чтобы экономить токены, но сохранять суть. Хранится в **SQLite**.
- **Долгосрочная память (Long-Term):** Вся история доступна через специальные **инструменты поиска** (`recall_memory`), что позволяет агенту "вспоминать" события по запросу.
- **Профиль пользователя (User Profile):** Факты о пользователе, его предпочтениях и стиле общения хранятся в **векторной базе данных (LanceDB)**. При каждом запросе ищутся наиболее релевантные факты, которые добавляются в промпт.

### D. Система Нейронов (Neurons)
- **Концепция:** "Нейрон" — это абстракция над любой AI-моделью (LLM, STT, TTS, Vision).
- **Гибкость:** Пользователь может настраивать, какую модель использовать для какой задачи (например, быструю модель для суммаризации и мощную для принятия решений).
- **Провайдеры:** Поддерживаются облачные API (OpenAI, Gemini) и локальные модели (Ollama, LlamaCPP).
- **`hai-core`:** Специальный сервис, который предоставляет унифицированный доступ к нейронам для всех плагинов, скрывая сложность работы с API-ключами.

### E. Система Плагинов (MCP)
- **MCP (Model-Component Protocol):** Протокол на основе `stdio` или `http`, позволяющий безопасно подключать внешние инструменты.
- **Изоляция:** Каждый плагин работает в отдельном процессе, что защищает ядро от сбоев.
- **Типы плагинов:**
    - **Tools:** Активные действия (поиск в вебе, работа с файлами).
    - **Snapshots:** Сбор данных о состоянии системы в реальном времени (например, URL активной вкладки).
    - **Watchers:** Фоновые процессы для отслеживания событий (активность AFK, изменения в файловой системе).

### F. Голосовая система (LiveKit)
- **Стек:** LiveKit (WebRTC-сервер), Python-воркер.
- **Low Latency:** Обеспечивает быструю передачу голоса для естественного диалога.
- **Компоненты воркера:**
    - **VAD (Voice Activity Detection):** Определяет, когда пользователь говорит.
    - **STT (Speech-to-Text):** Преобразует речь в текст (Whisper, Deepgram).
    - **TTS (Text-to-Speech):** Синтезирует ответ голосом (Piper, ElevenLabs).

## 4. Ключевые технические решения

1.  **Local-First и Приватность:** Все данные пользователя (история, профиль, активность) хранятся на его компьютере. Облачные сервисы используются только для инференса моделей (если выбраны).
2.  **Распределенная архитектура:** Разделение на независимые процессы (Backend, LiveKit, ActivityWatch) повышает стабильность. Сбой в одном компоненте (например, в голосовом движке) не приведет к падению всего ассистента.
3.  **Гибридная архитектура памяти:** Комбинация **векторного поиска** (для семантически релевантных фактов) и **суммаризации** (для хронологического контекста) позволяет агенту иметь глубокое и разностороннее понимание ситуации.
4.  **Протокол MCP:** Дает неограниченные возможности для расширения. Любой разработчик может написать инструмент на любом языке и безопасно интегрировать его в систему.
