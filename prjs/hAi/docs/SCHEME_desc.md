# Описание Архитектуры hAi

Этот документ описывает высокоуровневую архитектуру приложения hAi, представленную на схеме `SCHEME.canvas`. Система построена по модульному принципу, где Electron отвечает за UI, а Python-бэкенд управляет логикой, AI-агентами и интеграциями.

## 1. Frontend (Electron)
Пользовательский интерфейс и точка входа для десктопного приложения.

*   **Electron Main (`hey-electron/main.js`)**:
    *   Управляет жизненным циклом приложения, окнами и треем.
    *   Отвечает за глобальные шорткаты, нативные меню и автообновление.
    *   Является "родителем" для бэкенд-процесса (Orchestrator), запуская и останавливая его.
    *   Обеспечивает IPC (Inter-Process Communication) мост для рендерера.
*   **Renderer (React) (`hey-electron/src/`)**:
    *   Реализует визуальную часть: оверлей чата, страницу настроек, компоненты интерфейса.
    *   Взаимодействует с системными API только через безопасный `preload.js`.
*   **IPC Bridge (`preload.js`)**:
    *   Прослойка безопасности, экспортирующая конкретные методы (`api.settingsAPI`, `api.overlay`) в глобальный контекст браузера для общения с Main процессом.

## 2. Orchestrator (Backend/main.py)
Главный управляющий скрипт Python-части.
*   Запускается процессом Electron Main.
*   Отвечает за запуск и контроль жизненного цикла всех микросервисов бэкенда:
    *   **FastAPI Server**: Основной сервер логики.
    *   **LiveKit Server**: Бинарный сервер для WebRTC (голос).
    *   **LiveKit Worker**: Python-воркер для обработки голоса.
    *   **ActivityWatch Server**: Сервер мониторинга активности.
*   Обеспечивает корректное завершение всех процессов при выходе.

## 3. Backend (FastAPI)
Ядро логики приложения.

*   **FastAPI Application (`Backend/longchain_app.py`)**:
    *   Инициализирует приложение, подключает middleware (CORS) и статику.
    *   Управляет жизненным циклом MCP-менеджера и базы данных.
*   **API Routers (`Backend/api/`)**:
    *   `chat_router`: Эндпоинты для отправки сообщений и стриминга ответов LLM.
    *   `settings_router`: CRUD операции для настроек, управление моделями и плагинами.
    *   `state_router`: Обмен событиями между бэкендом и UI (например, уведомления о статусе).
*   **Chat Logic (`_unified_chat_processing_stream`)**:
    *   Основной конвейер обработки запроса: ожидание снимка состояния -> загрузка профиля и памяти -> сборка контекста -> запуск агента LangChain.
*   **Memory System (`Backend/modules/memory/`)**:
    *   Реализует логику долгосрочной и краткосрочной памяти.
    *   Включает суммаризатор (`summarizer.py`) для сжатия истории диалогов и активности.
*   **Plugin Loader**:
    *   Динамически загружает Core-плагины (Tools, Snapshots, Watchers) из директории `Backend/plugins/`.

## 4. MCP System (Model-Component Protocol)
Система для подключения внешних и внутренних инструментов через стандартизированный протокол.

*   **MCP Manager**: Управляет конфигурацией и соединениями с MCP-серверами (stdio, http, sse).
*   **hai-core (`Backend/hai_mcp/core/`)**: Внутренний MCP-сервер, который проксирует доступ к системным ресурсам (БД, Настройки, RAG) для других плагинов.
*   **External Plugins**: Изолированные процессы, предоставляющие дополнительные инструменты агенту.

## 5. Voice System (LiveKit)
Подсистема для голосового общения с низкой задержкой.

*   **LiveKit Server**: Внешний бинарный файл, обеспечивающий транспорт аудио (WebRTC SFU).
*   **Voice Agent Worker (`Backend/livekit/worker.py`)**:
    *   Подключается к комнате LiveKit как участник.
    *   **VAD (Silero)**: Определение активности голоса.
    *   **STT (Whisper/Deepgram)**: Перевод речи в текст.
    *   **LLM Wrapper**: Обработка текстовых команд (с поддержкой Wake Word).
    *   **TTS (Piper/Silero/OpenAI)**: Синтез речи из ответа LLM.

## 6. Activity Monitoring (ActivityWatch)
Система сбора контекста действий пользователя для памяти агента.

*   **ActivityWatch Server**: Хранилище временных рядов (buckets) для событий.
*   **Watchers (`Backend/plugins/watchers/`)**: Агенты, отслеживающие конкретные метрики (активное окно, простой, файловая система) и отправляющие их на сервер.

## 7. Data & Configuration
*   **SQLite Database (`Backend/db.sqlite`)**:
    *   Единое хранилище для истории сообщений, чанков суммаризации и статистики использования токенов.
*   **User Settings (`Backend/user_settings.json`)**:
    *   JSON-файл с конфигурацией пользователя (API ключи, выбранные модели, настройки плагинов).
*   **AI Model Providers**:
    *   Абстракция над внешними API (OpenAI, Anthropic, Google) и локальными моделями (Ollama, LlamaCpp), предоставляющая унифицированный интерфейс для "Нейронов".
