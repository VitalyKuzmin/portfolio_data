# Архитектура Нейронов

Этот документ описывает архитектуру "Нейронов" в приложении, их типы, источники данных и то, как они интегрированы в систему.

## 1. Что такое "Нейрон"?

"Нейрон" — это абстракция над AI-агентом, выполняющим определенную функцию. Это может быть большая языковая модель (LLM) для принятия решений, модель для распознавания речи (STT), синтеза речи (TTS) или специализированная модель для анализа изображений.

Раздел "Neurons" в настройках приложения является центральным местом для конфигурации всех этих агентов.

## 2. Источники данных и Провайдеры (`Backend/models_db`)

Директория `Backend/models_db` содержит JSON-файлы, которые служат реестрами доступных провайдеров и моделей для различных **Типов Нейронов**. Эти реестры используются для заполнения выпадающих списков в интерфейсе настроек.

### 2.1. Реестры для LLM-нейронов (`Text to Text`, `Text + Image to Text`, `Computer Vision`)

-   **`registry.text.json`**: Список текстовых LLM от различных провайдеров (например, OpenAI, Google, Anthropic), полученный с OpenRouter.
-   **`registry.vision.json`**: Список мультимодальных LLM, способных обрабатывать изображения, также с OpenRouter.
-   **`providers.text.json`**: Содержит конфигурации API для LLM-провайдеров, такие как URL для получения ключей API и названия пакетов.

### 2.2. Реестры для STT/TTS-нейронов (`Audio to Text`, `Text to Audio`)

-   **`registry.stt.json` / `registry.tts.json`**: Списки провайдеров STT и TTS (например, ElevenLabs, Deepgram).
-   **`providers.stt.json` / `providers.tts.json`**: Конфигурации API для провайдеров STT/TTS.
-   **`registry.whisper.json`**: Список доступных локальных моделей WhisperCPP.
-   **`registry.piper.json`**: Список доступных локальных моделей и голосов Piper TTS.

### 2.3. Скрипты для обновления реестров

-   `build_registry_text.py`: Загружает и обрабатывает список моделей с **OpenRouter API**, разделяя их на текстовые (`registry.text.json`) и vision (`registry.vision.json`).
-   `build_registry_stt_tts.py`: Собирает каталоги STT/TTS провайдеров (ElevenLabs, Deepgram, Cartesia, Google, OpenAI), используя их API и файлы-оверрайды.
-   `build_livekit.py`: Парсит документацию LiveKit для получения информации о STT/TTS провайдерах и генерирует `providers.stt.json` и `providers.tts.json`.
-   `build_whispercpp.py`: Генерирует `registry.whisper.json`, собирая информацию о доступных моделях из констант `pywhispercpp` и репозитория OpenAI.
-   `build_piper.py`: Загружает `voices.json` с HuggingFace для создания `registry.piper.json`, содержащего информацию о голосах и моделях Piper.

## 3. Типы Нейронов

Каждый нейрон может быть одного из следующих типов, который определяет его основную функцию и доступные настройки.

-   **Text to Text**: Стандартная языковая модель, принимающая текст и возвращающая текст. Используется для большинства задач.
    -   **Источники данных**: `registry.text.json`, `providers.text.json`.

-   **Text + Image to Text**: Мультимодальная модель, способная анализировать текст и изображения для генерации текстового ответа.
    -   **Источники данных**: `registry.vision.json`, `providers.text.json`.

-   **Computer Vision**: Специализированная модель для задач компьютерного зрения, таких как взаимодействие с UI.
    -   **Источники данных**: `registry.computer_vision.json`, `providers.text.json`.

-   **Audio to Text**: Модель распознавания речи (STT), преобразующая аудио в текст.
    -   **Источники данных**: `registry.stt.json`, `providers.stt.json`, `registry.whisper.json`.

-   **Text to Audio**: Модель синтеза речи (TTS), преобразующая текст в аудио.
    -   **Источники данных**: `registry.tts.json`, `providers.tts.json`, `registry.piper.json`.

-   **Custom**: Позволяет настроить любой OpenAI-совместимый эндпоинт, указывая `base_url` и `api_key`.
    -   **Источник данных**: Настраивается пользователем вручную.

-   **Ollama**: Для запуска моделей локально через Ollama.
    -   **Источник данных**: Настраивается пользователем вручную.

-   **Llamacpp**: Для запуска моделей локально через `llama.cpp`.
    -   **Источник данных**: Настраивается пользователем вручную.

## 4. Core и Custom Нейроны

Нейроны делятся на две категории: "Core" (встроенные) и "Custom" (пользовательские).

### 4.1. Core Нейроны

Это предустановленные нейроны, каждый из которых выполняет ключевую роль в приложении. Их настройки хранятся в соответствующих секциях `user_settings.json`.

-   **Director**: Главный управляющий агент. Отвечает за анализ запросов пользователя и принятие решений о вызове инструментов.
    -   **Тип по умолчанию**: `Text to Text`.

-   **Computer Use**: Агент, используемый инструментом `run_ui_scenario` для выполнения сложных взаимодействий с графическим интерфейсом.
    -   **Тип по умолчанию**: `Computer Vision`.

-   **Image Recognition**: Агент, используемый инструментом `screenshot_view` для анализа содержимого экрана и ответа на вопросы о нем.
    -   **Тип по умолчанию**: `Text + Image to Text`.

-   **Speech To Text**: Агент для распознавания речи, используемый в разделе "Hearing".
    -   **Тип по умолчанию**: `Audio to Text`.

-   **Text to Speech**: Агент для синтеза речи, используемый в разделе "Speaking".
    -   **Тип по умолчанию**: `Text to Audio`.

-   **Text Summary**: Агент, отвечающий за суммирование информации.
    -   **Тип по умолчанию**: `Text to Text`.
    -   **Использование**:
        -   **Summary Memory**: Суммирует длинные диалоги для экономии токенов.
        -   **Web Searching**: Суммирует содержимое веб-страниц.

### 4.2. Custom Нейроны

Пользователи могут создавать свои собственные нейроны через интерфейс настроек.

-   **Создание**: При нажатии на "+" в секции "Custom" создается новый нейрон с именем по умолчанию (`Custom Neuron 1`, `Custom Neuron 2` и т.д.).
-   **Хранение**: Конфигурации пользовательских нейронов хранятся в `user_settings.json` в объекте `neurons.custom`. Каждый нейрон имеет уникальный ID в качестве ключа.
-   **Гибкость**: Пользовательские нейроны могут быть любого типа, что позволяет гибко настраивать и интегрировать различные модели и сервисы.
