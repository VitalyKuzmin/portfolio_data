# Сборка проекта

Этот документ описывает процесс сборки приложения Hey Assistant для macOS.

## Требования

*   macOS (Apple Silicon или Intel)
*   Node.js (v18+)
*   npm

## Пошаговая инструкция

1.  **Подготовка окружения**

    Перейдите в папку с Electron-приложением:
    ```bash
    cd hey-electron
    ```

2.  **Загрузка изолированного Python**

    Для работы приложения на компьютерах пользователей нам нужна встроенная версия Python. Скачаем её (пример для Apple Silicon / M-series):

    ```bash
    # Для Apple Silicon (M1/M2/...)
    curl -L -o python.tar.gz https://github.com/indygreg/python-build-standalone/releases/download/20240415/cpython-3.11.9+20240415-aarch64-apple-darwin-install_only.tar.gz

    # Распаковка
    tar -xzf python.tar.gz && rm python.tar.gz
    ```

    > **Важно:** Если вы собираете на Intel Mac, используйте соответствующую ссылку на `x86_64-apple-darwin` релиз с [python-build-standalone](https://github.com/indygreg/python-build-standalone/releases).

3.  **Установка зависимостей бэкенда**

    Установим все необходимые Python-библиотеки прямо в скачанную версию Python. Это гарантирует, что у пользователя будут все нужные пакеты.

    ```bash
    ./python/bin/python3 -m pip install -e ../Backend
    ```

4.  **Сборка приложения**

    Установите JS-зависимости и запустите сборку:

    ```bash
    npm install
    npm run build
    ```

## Результат

После завершения процесса в папке `hey-electron/dist/` появятся:
*   `.dmg` файл — установщик для распространения.
*   `.app` файл — исполняемое приложение (в подпапке `mac-arm64` или `mac`).

## Структура сборки

В `package.json` настроена секция `extraResources`, которая копирует:
*   Папку `python` (с установленными библиотеками) -> в ресурсы приложения.
*   Папку `Backend` (код бэкенда) -> в ресурсы приложения.

При запуске `main.js` определяет, запущено ли приложение в упакованном виде, и использует соответствующий путь к Python и бэкенду.

## 5. Подпись приложения (Code Signing)

### Проблема

При запуске собранного, но не подписанного приложения, macOS будет постоянно запрашивать разрешения (на доступ к микрофону, специальным возможностям и т.д.) при каждом запуске, даже если вы их уже предоставили. Это происходит потому, что система безопасности Gatekeeper не может проверить подлинность приложения.

### Решения

#### 1. Для локального тестирования (Ad-hoc)

Вы можете "самоподписать" приложение после сборки. Это может помочь системе "запомнить" выданные разрешения на вашем компьютере. Выполняйте эту команду из корневой папки проекта после каждой сборки:

```bash
codesign --force --deep --sign - hey-electron/dist/mac-arm64/hAi.app
```
*Примечание: Замените `hAi.app` на актуальное имя вашего `.app` файла, если оно отличается.*

#### 2. Для распространения (Apple Developer ID)

Для того чтобы другие пользователи могли без проблем использовать ваше приложение, его необходимо подписать официальным сертификатом разработчика.

-   **Что нужно:** Сертификат "Developer ID Application".
-   **Как получить:** Вступить в программу **Apple Developer Program**.
-   **Стоимость:** **$99 в год**.

Подписанное приложение корректно проходит проверки Gatekeeper, и система сохраняет все выданные ему разрешения.

## 6. Продвинутые опции сборки

### Скрытие исходного кода Python (Обфускация)

Чтобы защитить исходный код бэкенда от простого просмотра, его можно "заморозить" — упаковать в один исполняемый файл с помощью **PyInstaller**.

**План действий:**

1.  **Установить PyInstaller** в изолированный Python:
    ```bash
    ./python/bin/python3 -m pip install pyinstaller
    ```
2.  **Создать скрипт сборки** (например, `hey-electron/build_backend.py`), который будет запускать PyInstaller с нужными параметрами для `Backend/main.py`.
3.  **Изменить `hey-electron/package.json`**:
    *   В секцию `extraResources` вместо копирования всей папки `Backend` добавить копирование одного собранного исполняемого файла (например, из `Backend/dist/main`).
    *   Удалить из `extraResources` копирование папки `python`, так как PyInstaller включит все необходимое в исполняемый файл.
4.  **Изменить `hey-electron/main.js`**:
    *   Изменить логику `startBackend`, чтобы она запускала не `python ... main.py`, а напрямую исполняемый файл бэкенда.
5.  **Изменить код бэкенда**: Внести правки в код (например, в `common/user_settings.py`), чтобы он мог корректно находить файлы данных (такие как `.json` файлы) после "заморозки".

### Сборка под Windows

Сборка под Windows возможна и требует выполнения процесса на машине с Windows.

**Необходимые изменения:**

1.  **Кроссплатформенная очистка:** В `hey-electron/package.json` команда `prebuild` использует `rm -rf dist`. Для Windows ее нужно заменить на кроссплатформенный аналог. Сначала установите пакет `rimraf`:
    ```bash
    npm install rimraf --save-dev
    ```
    Затем измените скрипт:
    ```json
    "prebuild": "rimraf dist && npm run build:react"
    ```
2.  **Загрузка Python для Windows:** Вместо версии для `darwin` скачайте архив для Windows со страницы [python-build-standalone](https://github.com/indygreg/python-build-standalone/releases) (например, `...-pc-windows-msvc-install_only.tar.gz`).
3.  **Команды:** Команды установки и сборки останутся теми же по сути, но пути к исполняемым файлам изменятся на Windows-стиль (например, `python\\python.exe -m pip install ...`).
