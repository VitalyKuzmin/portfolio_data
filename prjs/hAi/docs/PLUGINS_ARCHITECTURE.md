# Архитектура Плагинов

Этот документ описывает архитектуру плагинов в приложении, их типы и способы взаимодействия с основной системой.

## 1. Обзор архитектуры

Система плагинов состоит из двух основных категорий:

1.  **Core Plugins (Python):** Внутренние плагины, поставляемые с приложением. Они написаны на Python и работают в том же процессе, что и основной бэкенд. Их зависимости управляются централизованно для обеспечения стабильности.
2.  **MCP Plugins (Внешние):** Пользовательские плагины, работающие как отдельные, изолированные процессы. Они могут быть написаны на любом языке и взаимодействуют с основным приложением через **Model-Component Protocol (MCP)**.

### Схема взаимодействия:

```
[ Electron UI ] <--> [ Python Backend (FastAPI) ] <--> [ MCP Manager ]
                                |                             |
                                |                             +-- [ MCP Plugin A ]
                                |                             +-- [ MCP Plugin B ]
                                |
                           [ Core Plugins (Python) ]
```

-   **MCP Manager:** Компонент Python-бэкенда, отвечающий за жизненный цикл внешних MCP-плагинов (запуск, остановка, мониторинг).
-   **`hai-core`:** Специальный внутренний MCP-сервер, предоставляющий базовые сервисы (хранилище, RAG, настройки, нейроны) для всех плагинов.

## 2. Core Plugins

-   **Расположение:** `Backend/plugins/`
-   **Загрузка:** Загружаются при старте приложения с помощью `plugin_loader.py`.
-   **Взаимодействие:** Могут напрямую вызывать функции Python или использовать клиент `plugins.core_client` для доступа к сервисам `hai-core`.

## 3. MCP Plugins

-   **Концепция:** Изолированные процессы, общающиеся с приложением через `stdio` или `http`.
-   **Конфигурация:** Регистрируются в `mcp_*.json` файлах в директории `Backend/hai_mcp/manager/`.
-   **Взаимодействие:** Все вызовы к MCP-плагинам и от них проходят через **MCP Manager**.
-   **Разработка:** Подробное руководство по созданию MCP-плагинов доступно в файле [MCP_INSTRUCTIONS.md](../MCP_INSTRUCTIONS.md).

## 4. Core Services (`hai-core`)

`hai-core` — это сервисная шина, доступная всем плагинам через MCP. Она предоставляет унифицированный API для ключевых функций:
-   **Store:** Хранение и получение данных.
-   **RAG:** Поиск по локальной базе знаний.
-   **Settings:** Доступ к настройкам приложения.
-   **Snapshots & Watchers:** Доступ к данным о состоянии системы.
-   **Neurons:** Выполнение запросов к AI-моделям (LLM, STT, TTS и др.), абстрагируя детали их реализации.

## 5. Интеграция с системой Нейронов

Плагины (как Core, так и MCP) могут использовать "Нейроны" — абстракции над AI-моделями.

1.  **Декларация в `settings.json`:** Плагин может объявить, что ему нужен нейрон, добавив в свой файл `settings.json` поле с типом `"neuron_selector"`. Это создаст в UI выпадающий список для выбора нейрона.
    ```json
    {
      "key": "agent",
      "label": "Agent",
      "type": "neuron_selector",
      "neuron_type": "text_to_text"
    }
    ```
2.  **Использование в коде:**
    -   Плагин получает полную конфигурацию выбранного пользователем нейрона из своих настроек.
    -   Для выполнения запроса плагин вызывает инструмент `core_neurons_call` из `hai-core`, передавая ему конфигурацию нейрона и промпт.
    -   `hai-core` берет на себя создание нужной модели, управление API-ключами и выполнение запроса.

Этот подход позволяет разработчикам плагинов легко интегрировать любую AI-модель, поддерживаемую системой, не вникая в детали ее API.
