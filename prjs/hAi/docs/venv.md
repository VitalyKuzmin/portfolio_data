# Виртуальные окружения (venv) и безопасность PythonCodeExecutor

Этот документ объясняет, что такое виртуальные окружения в Python и как они используются в `hAi` для безопасного выполнения кода.

## 1. Что такое виртуальное окружение (`venv`)?

**Виртуальное окружение** в Python — это изолированная среда, которая имеет свой собственный интерпретатор Python и набор установленных библиотек.

**Ключевые цели:**

-   **Изоляция зависимостей:** Представьте, что Проект А требует библиотеку `requests` версии 1.0, а Проект Б — `requests` версии 2.0. Если устанавливать их глобально, возникнет конфликт. Виртуальные окружения решают эту проблему, позволяя каждому проекту иметь свой собственный, независимый набор библиотек.
-   **Воспроизводимость:** Файл `requirements.txt` в проекте позволяет точно воссоздать его окружение на другой машине, гарантируя, что код будет работать одинаково.
-   **Чистота системы:** Системный Python остается "чистым", без множества проект-специфичных библиотек.

В `hAi` мы идем на шаг дальше: мы не просто используем `venv`, а поставляем **полностью изолированный, портативный дистрибутив Python** вместе с приложением. Это гарантирует, что `hAi` никак не зависит от того, установлен ли Python на компьютере пользователя, и не конфликтует с ним.

## 2. Как `hAi` использует изолированный Python для `PythonCodeExecutor`?

Инструмент `PythonCodeExecutor` позволяет AI-агенту писать и выполнять Python-скрипты для автоматизации задач на компьютере пользователя (работа с файлами, браузером и т.д.).

Когда агент решает использовать этот инструмент:
1.  Он генерирует Python-код.
2.  Этот код выполняется с помощью **встроенного, изолированного интерпретатора Python**, который поставляется вместе с `hAi`.
3.  Код имеет доступ к специальному объекту `computer`, который предоставляет безопасные функции для взаимодействия с ОС (например, `computer.files.search_files(...)`).

## 3. Почему этот подход безопасен?

Безопасность выполнения кода, сгенерированного LLM, является ключевым приоритетом. Вот почему наш подход минимизирует риски:

1.  **Полная изоляция от системного Python:**
    -   Агент не может установить вредоносную или конфликтующую библиотеку в глобальный Python пользователя. Любая установка (`pip install`) произойдет внутри "песочницы" `hAi` и не затронет систему.
    -   Мы гарантируем, что у агента именно та версия Python и библиотек, с которой мы тестировали систему.

2.  **Отсутствие повышенных привилегий:**
    -   Процесс, выполняющий Python-скрипт, наследует права от основного приложения `hAi`.
    -   Поскольку `hAi` не запрашивает права администратора (`sudo`), сгенерированный код также не может выполнять действия, требующие повышенных привилегий (например, изменять системные файлы).

3.  **Контролируемый API для взаимодействия с ОС:**
    -   Основной способ взаимодействия с компьютером для агента — это использование предопределенных, безопасных функций объекта `computer` (например, `computer.browser.open_url`).
    -   Хотя агент *может* сгенерировать и выполнить произвольный код (например, через `os.system`), его промпт и "обучение" нацелены на использование безопасного API. Это снижает вероятность выполнения деструктивных команд.

4.  **Прозрачность:**
    -   В будущем планируется добавить механизм, который будет показывать пользователю сгенерированный код перед выполнением, давая ему возможность подтвердить или отклонить операцию.

Таким образом, использование встроенного, изолированного Python-окружения создает надежную "песочницу", которая защищает систему пользователя и обеспечивает стабильную и предсказуемую среду для работы AI-агента.
